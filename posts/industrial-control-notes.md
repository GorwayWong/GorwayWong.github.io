# 从 L2 工控系统到后端工程

在钢铁产线的 L2 系统里，服务端程序并不是单纯处理 HTTP 请求，而是要持续跟踪生产过程中的物料、设备状态和上下游系统消息。

这类系统给我的工程启发主要有三点。

## 状态必须清晰

工控系统里一个钢卷或钢板的位置变化会影响后续很多动作。状态机设计、事件触发顺序和异常恢复路径都需要非常明确。

## 通信边界要稳定

L2 系统常通过 TCP Socket 与 PLC、MES 等系统通信。报文结构、字段含义和超时处理一旦不稳定，问题会直接反映到生产节奏上。

## 数据要能追溯

实绩上报、校准记录、过程参数都需要落库。服务端不仅要把流程跑通，还要让事后的查询、追溯和排障有据可查。

这些经验迁移到互联网后端开发中，依然对应着接口契约、状态一致性、可观测性和数据建模。
